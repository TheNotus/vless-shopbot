# ChronoLog

*Минималистичное, self-hosted веб-приложение для учета рабочего времени.*

## Краткое описание

ChronoLog — это простое и эффективное решение для фрилансеров, удаленных команд и всех, кому нужен удобный инструмент для отслеживания времени, затраченного на различные проекты и задачи. Приложение ориентировано на минимализм и простоту использования, позволяя вам сосредоточиться на работе, а не на управлении инструментами.

## Ключевые особенности

*   **Логирование времени:** Простое создание, редактирование и удаление записей о времени.
*   **Информативный дашборд:** Визуализация данных о затраченном времени для анализа продуктивности.
*   **Экспорт данных:** Возможность выгрузки отчетов в формате CSV для дальнейшей обработки или отчетности.
*   **Self-hosted:** Полный контроль над вашими данными. Вы можете развернуть приложение на собственном сервере.

## Технологический стек

*   **Бэкенд:** Python 3.11, FastAPI
*   **Фронтенд:** TypeScript, React, Vite
*   **База данных:** PostgreSQL
*   **Управление зависимостями (бэкенд):** Poetry
*   **Управление зависимостями (фронтенд):** NPM

## Начало работы (Getting Started)

Для локального запуска проекта выполните следующие шаги:

### 1. Клонирование репозитория

```bash
git clone https://github.com/your-username/chronolog.git
cd chronolog
```

### 2. Настройка и запуск бэкенда

Убедитесь, что у вас установлен [Poetry](https://python-poetry.org/docs/#installation).

```bash
# Установка зависимостей
poetry install

# Запуск сервера для разработки
poetry run uvicorn app.main:app --reload
```
Сервер бэкенда будет доступен по адресу `http://127.0.0.1:8000`.

### 3. Настройка и запуск фронтенда

Перейдите в директорию `web` в новом окне терминала.

```bash
cd web

# Установка зависимостей
npm install

# Запуск сервера для разработки
npm run dev
```
Веб-интерфейс будет доступен по адресу, указанному в выводе команды (обычно `http://localhost:5173`).
